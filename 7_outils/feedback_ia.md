# Template — Feedback IA (Claude)

> Comment demander à Claude une critique structurée sur ton manuscrit.
> Copie-colle le bloc adapté selon ce que tu veux faire critiquer.

---

## Demande type — Critique d'un chapitre

```
Lis le fichier 5_ecriture/brouillon/chapitre_XX.md et fais une critique structurée selon :

1. Voix narrative — est-elle cohérente avec celle des chapitres précédents ?
2. Caractérisation — quel(s) personnage(s) sortent de leur lisibilité (principe 19) ?
3. Rythme — y a-t-il des moments où le lecteur saturerait (principe 35) ?
4. Antériorité — tout ce qui se passe découle-t-il de ce qui a été écrit avant (principe 25) ?
5. Faiblesse fondatrice — est-ce qu'au moins une scène attaque la faiblesse du protagoniste ?
6. Coupes possibles — quelles scènes/paragraphes pourraient être coupés sans perte ?

Note sur 5 chacun de ces 6 axes, et donne-moi 3 actions concrètes à appliquer en priorité.
```

## Demande type — Critique d'un acte complet

```
Lis tous les chapitres de l'Acte X dans 5_ecriture/brouillon/.

1. Passe la checklist 6_revision/checklist_acteX.md mentalement
2. Note ce qui manque par rapport à la trame planifiée dans 3_trame/acteX.md
3. Identifie les 3 plus gros problèmes structurels
4. Identifie les 3 plus grandes forces

Donne-moi un verdict sur 50 selon la grille 7_outils/grille_evaluation.md.
```

## Demande type — Critique du manuscrit entier

```
Lis tous les chapitres dans 5_ecriture/brouillon/.

Applique la grille 7_outils/grille_evaluation.md complète, axe par axe.

Pour chaque axe, donne :
- Score brut /5
- Score pondéré
- Justification (1-2 phrases)
- Action prioritaire si score < 3

Total final /70 + verdict du niveau.

Identifie aussi les violations majeures des 40 principes du guide.
```

## Demande type — Cohérence d'un personnage

```
Vérifie la cohérence du personnage [NOM] :
1. Lis 1_personnages/[nom].md (sa fiche)
2. Identifie toutes ses apparitions dans 5_ecriture/brouillon/
3. Liste les scènes où son comportement diverge de sa psychologie établie
4. Note s'il a un arc détectable ou s'il reste statique

Verdict : passe-t-il le test de robustesse (Section VI.C/D/E) ?
```

## Demande type — Vérification des piliers narratifs

```
Liste tous les piliers narratifs plantés dans le manuscrit :
- Mystères ouverts
- Promesses faites au lecteur
- Compte à rebours visible
- Questions soulevées

Pour chacun : est-il payé à la fin du manuscrit ?

Identifie les piliers OUBLIÉS (plantés mais jamais payés — violation principe 18 et 25).
```

## Demande type — Détection des pièges classiques

```
Cherche dans le manuscrit les 8 pièges documentés dans 7_outils/ref_pieges.md :

1. Perso pivot sous-construit ?
2. Bascule psychologique non-dramatisée ?
3. Rival sans introspection précoce ?
4. Mentor-coach sans transmission ?
5. Révélation trop tardive ?
6. Lore mal expliqué avant moment-clé ?
7. Romance intermittente ?
8. Rejet mal justifié ?

Pour chaque piège trouvé : où précisément, et comment le réparer.
```

## Comment lire les retours de Claude

### Hiérarchie de confiance des retours IA

1. **Les retours TECHNIQUES** (cohérence factuelle, principe violé, faute de structure) → fiables, à appliquer
2. **Les retours STRUCTURELS** (pacing, équilibre, manque) → fiables, à considérer sérieusement
3. **Les retours STYLISTIQUES** (formulation, mot choisi) → souvent bons mais pas absolus
4. **Les retours ÉMOTIONNELS** (Claude n'a pas d'émotion réelle) → utile comme grille mais pas comme vérité

### Quand faire confiance à Claude > bêta-lecteur humain

- Vérification de cohérence factuelle / timeline
- Application stricte d'un principe précis
- Détection de violations qui passent inaperçues à un humain pressé

### Quand faire confiance à un humain > Claude

- Émotion ressentie en lisant
- Si le livre "marche" globalement
- Si le rythme accroche ou non
- Pour TON style propre (Claude tend vers le générique)

> **Conclusion** : croise toujours feedback IA + feedback humain. Aucun n'est suffisant seul.
